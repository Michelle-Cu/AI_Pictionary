## 遊戲配置

| 項目 | 規格說明 |
|------|-------------|
| 對抗規模 | 小隊競爭（每隊 12 人，細分為 3 個 4 人接龍小組） |
| 題目規格 | 預先由關主生成的高品質 AI 圖片（具備明確主體、環境與強烈風格） |
| 評分判定 | 50%分數取決於是否有符合特定關鍵字、50%取決於圖片比對系統相似度 |
| 生成與判定工具 | 透過 Gemini 進行圖片生成，並以 Python (Gradio 套件)架設本地端網頁計分伺服器 |

## 遊戲規則

1. **比賽前教學**：
* 進行 5-10 分鐘的 Prompt 基礎框架教學（主體 + 環境 + 風格），提供英文提示詞小抄 (Cheat Sheet) 作為輔助工具。
2. **Prompt 接龍創作**：
* 每小隊 12 人分為 3 組，每組 4 人進行接龍協作撰寫 Prompt。
* 可以讓組內自行分配誰負責寫哪一部分（例如四個人負責寫圖片角色的 prompt、四個人負責寫圖片風格的 prompt、四個人負責寫背景細節的 prompt）。
3. **評分**：
* 小隊將接龍寫出的 Prompt 輸入 Gemini 生成圖片。
* 將生成的圖片上傳至 Gradio 計分系統，系統會自動與「標準原圖」進行比對並給出分數。
* 分數較高的小隊獲勝。

---

## 技術實踐架構

1. 前端互動介面： 
* 使用 Python 的 Gradio 套件架設 Web 介面。開啟 share=True 功能生成公開網址，讓營隊現場操作的流暢方便。

2. 後端語義比對機制：
* 採用 openai/clip-vit-base-patch32 預訓練模型。將標準原圖與學員生成圖輸入模型，抓取其底層的特徵張量 (Tensor)，並進行 L2 正規化。
* 計算兩向量的內積 $\text{Similarity} = \vec{v}_{ref} \cdot \vec{v}_{team}$，以此作為語義與風格契合度的客觀標準。

## 教學目標

- **學習精準寫 prompt**：讓學員理解與生成式 AI 溝通的邏輯，體會精準描述細節（材質、風格、視角等等）的重要性。

## 可能問題與對策

- **分數比對**：目前 CLIP 模型只有嘗試過一組照片，不確定是不是在大部分照片中都能有很好的效果。因此目前打算將圖片比對分數佔 50% 分數，另外 50% 由我們自行訂定每個照片的 prompt 應該要寫到的關鍵字（例如在下面這段 prompt 中，應該要寫到 "Cartoon", "fox", "mouse", "snow", "chase/ run after/ catch" 等關鍵字）。

A cartoon image with a brown fox in the snow, there are mountains with snow caps in the backdrop, the fox is pouncing on the snow trying to catch a small grey mouse, there are fox footprints in the snow.

- **人數問題**：一組 12 個人很有可能只有一兩個人在寫 prompt，所以把每組拆成四個人一小組，給定他們負責撰寫的 prompt 內容，例如：四個人負責想怎麼寫形容圖片風格的提示詞、四個人負責想怎麼精準形容圖片角色以及情境、四個人負責想照片背景與細節的敘述。

- **語言問題**：不確定小隊員是否都能用英文寫 prompt，因此目前打算提供一些常見英文提示詞的 cheat sheet。（或是要改用中文 prompt??）
